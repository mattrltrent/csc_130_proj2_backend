Get random meal:


Try calling the random meal API:

Remember, you're going to the endpoint: website domain/ + random

https://pure-river-73157.herokuapp.com/random



Call this endpoint from your code: https://pure-river-73157.herokuapp.com/random

Notice, the route is: website domain/ + random

Here is an example call: let response = await fetch(https://pure-river-73157.herokuapp.com/random).

What kind of JSON is returned?

This returns an object:

Key Type Description
name String The name of the meal
image_source Link The link to an image of the meal.
video_source Link The link to a video of the meal being made.
tags List of strings Tags that tell what the meal is all about.
ingredients List of objects, seen in table below. Ingredients needed to make the meals.
recipe_source Link The link to the website the recipe is being sourced off of.

That inside houses a bunch of these ingredient objects:

Key Type Description
item String The actual item that is needed.
amount String The amount of "item" that is needed.

Get list of meals from a search query:


Try entering an example search query to test the search-query-based meal API:

Remember, you're going to the endpoint: website domain/ + meals/ + what you type in the box

https://pure-river-73157.herokuapp.com/meals/



Call this endpoint from your code: https://pure-river-73157.herokuapp.com/meals/YOUR_QUERY

Notice, the route is: website domain/ + meals/ + YOUR_QUERY

An example of YOUR_QUERY could be "car", which would return a list of recipes whose names contain "car" in it. This might includes "CARrot noodles" or "CARnival soup".

Here is an example call: let response = await fetch(https://pure-river-73157.herokuapp.com/meals/car).

What kind of JSON is returned?

This returns an object:

Key Type Description
length integer The number of meals returned for this search query.
meals List of objects, seen in table below. The actual individual meals.

That inside houses a bunch of these meal objects:

Key Type Description
name String The name of the meal.
image_source Link The link to an image of the meal.
video_source Link The link to a video of the meal being made.
tags List of strings Tags that tell what the meal is all about.
ingredients List of objects, seen in table below. Ingredients needed to make the meals.
recipe_source Link The link to the website the recipe is being sourced off of.

That inside houses a bunch of these ingredient objects:

Key Type Description
item String The actual item that is needed.
amount String The amount of "item" that is needed.
